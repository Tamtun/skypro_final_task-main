Git

Операции с файлами

  • rm [файл] - Удаляет конкретный файл из рабочей директории и индексирует его удаление
  • rm --cached [файл] - Убирает конкретный файл из контроля версий, но физически оставляет его на своём месте
  • mv [оригинальный файл] [новое имя] - Перемещает и переименовывает указанный файл, сразу индексируя его для последующего коммита

Игнорирование некоторых файлов

  • ls-files --others --ignored --exclude-standard - Список всех игнорируемых файлов в текущем проекте

Сохранение фрагментов

  • stash - Временно сохраняет все незафиксированные изменения отслеживаемых файлов
  • stash pop - Восстанавливает состояние ранее сохранённых версий файлов
  • stash list - Выводит список всех временных сохранений
  • stash drop - Сбрасывает последние временно сохранённыe изменения

Внесение изменений

  • status - Перечисляет все новые или изменённые файлы, которые нуждаются в фиксации
  • diff - Показывает различия по внесённым изменениям в ещё не проиндексированных файлах
  • add [файл] - Индексирует указанный файл для последующего коммита
  • diff --staged - Показывает различия между проиндексированной и последней зафиксированной версиями файлов
  • reset [файл] - Отменяет индексацию указанного файла, при этом сохраняет его содержимое
  • commit -m "[сообщение с описанием]" - Фиксирует проиндексированные изменения и сохраняет их в историю версий

Коллективная работа

  • branch - Список именованных веток коммитов с указанием выбранной ветки
  • branch [имя ветки] - Создаёт новую ветку
  • switch -c [имя ветки] - Переключается на выбранную ветку и обновляет рабочую директорию до её состояния
  • merge [имя ветки] - Вносит изменения указанной ветки в текущую ветку
  • branch -d [имя ветки] - Удаляет выбранную ветку

Просмотр истории

  • log - История коммитов для текущей ветки
  • log --follow [файл] - История изменений конкретного файла, включая его переименование
  • diff [первая ветка]...[вторая ветка] - Показывает разницу между содержанием коммитов двух веток
  • show [коммит] - Выводит информацию и показывает изменения в выбранном коммите

Откат коммитов

  • reset [коммит] - Отменяет все коммиты после заданного, оставляя все изменения в рабочей директории
  • reset --hard [коммит] - Сбрасывает всю историю вместе с состоянием рабочей директории до указанного коммита.

Синхронизация с удалённым репозиторием

  • fetch [удалённый репозиторий] - Скачивает всю историю из удалённого репозитория
  • [удалённый репозиторий]/[ветка] - Вносит изменения из ветки удалённого репозитория в текущую ветку локального репозитория
  • push [удалённый репозиторий] [ветка] - Загружает все изменения локальной ветки в удалённый репозиторий
  • pull - Загружает историю из удалённого репозитория и объединяет её с локальной. pull = fetch + merge
